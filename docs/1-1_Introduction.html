Introduction and History

Authors: Karl Brand, Elizabeth Ribble and S. Willemsen

Introduction

This chapter is here to get you oriented. If you are new to R, the main thing I want is for the environment to start feeling less mysterious: what R is, why people use it, what it feels like to type a few commands, and how RStudio fits into the picture. We will also look at packages and a few basic survival skills, because in real work the trick is rarely knowing everything in advance. It is knowing how to find your way when you do not.

What is R

R is a programming language and software environment that is mainly used for statistics, data analysis and creating graphics. It runs on all standard computing platforms, including Windows, MacOS and Linux. It is free and open source software, so everybody can download and use it without cost. It is also highly extensible, with thousands of packages available for statistical techniques, data manipulation and visualisation.

One of the success factors of R is its large and active user community, especially in statistics, epidemiology, bioinformatics and data science. This means that if you run into a problem, it is usually possible to find help. There are many forums, tutorials, blog posts and official help pages that can help you learn R and solve problems.

The R language is modular: base R has a lot of core functionality, but it can be extended by means of packages. Because of the popularity of R among statisticians, new statistical methods are often implemented in R very quickly, sometimes by the method’s original author. Nowadays there is an enormous number of extension packages. Their quality varies, although there is some quality control for packages distributed through the official channel CRAN.

Why learn R

There are many tools that can be used for data analysis, so it is reasonable to ask why we would spend time learning R.

  • It allows you to combine data cleaning, analysis, figures and reporting in one workflow.
  • It is script based, which means your work can be saved, checked, shared and repeated later.
  • It is widely used in research, so many examples and methods are already available.
  • It scales well from very simple calculations to advanced analyses.

For this course, an important advantage is that the same code can be rerun later. That makes your work more transparent and easier to check.

Your first 5 minutes in R

Before discussing installation details and history, it is useful to see what working in R actually feels like.

A first success

If you have R or RStudio open, try typing the following commands and running them one by one:

> 2 + 2
[1] 4
> sqrt(25)
[1] 5
> x <- 10
> x / 2
[1] 5

In just a few lines, you have already done three common things in R:

  • evaluated an expression
  • used a function
  • stored a result in a variable

That is the basic rhythm of working in R: type code, run it, inspect the result, and then build on it.

Common beginner worries

If you are new to programming, it is completely normal if the console, prompts and code syntax feel unfamiliar at first. You do not need to understand everything immediately. Early on, the goal is simply to become comfortable with trying small commands, reading the output, and correcting mistakes when needed.

History

Before R there was the statistical analysis program S, developed at Bell Laboratories in 1976 by John Chambers. It was developed with the purpose of being useful for interactive data analysis and programming. S was later commercialised as S-PLUS. S-PLUS contained some features that made working with the language easier, such as a graphical user interface (GUI). John Chambers is also a current member of the board of the R Foundation for Statistical Computing.

In 1991, Ross Ihaka and Robert Gentleman of the Department of Statistics, University of Auckland, New Zealand, created the S clone R as an open source alternative for academic use. Together with the R Core Team they released version 1 under the GNU Public License (GPL) in 2000. The name R probably derives from the first letter of the creators’ names. See also http://cran.r-project.org/doc/FAQ/R-FAQ.html#Why-is-R-named-R_003f.

Get it

Install it

To get R on your computer, you first need to install the core R application. You will need to download the version for your operating system and architecture.

  • For Linux: https://cran.r-project.org/bin/linux/

  • For Mac: https://cran.r-project.org/bin/macosx/

  • For Windows: https://cran.r-project.org/bin/windows/

Although R for Windows and Mac ships with a basic interface, and it is possible to execute code directly from a terminal under Linux, a fully featured editor or IDE makes your workflow much easier. A few common options are:

  • RStudio: https://www.rstudio.com/products/rstudio/

  • Visual Studio Code: https://code.visualstudio.com/

  • Positron: https://github.com/posit-dev/positron

  • Emacs/ESS: http://ess.r-project.org/index.php?Section=download

Here is a larger list of interfaces for R: https://en.wikipedia.org/wiki/R_(programming_language)#Interfaces

How do we use R

Working with R

You type commands, also known as expressions, into a text editor or terminal and send them to R for evaluation. R evaluates the command and then prints the result, if there is one.

The simplest expressions equal simple calculations, similar to those we could do on a hand calculator

>1 + 1
[1] 2
>1 - 1
[1] 0
>2 * 3
[1] 6
>2 / 3
[1] 0.6666667

Here we use > to denote the R prompt. In the console, this prompt indicates that R is ready for a new command. The lines underneath are the output given by R.

But we can also call functions. Most of the functionality in R comes from these functions. There are many functions we can use, including standard mathematical ones:

>cos(3.1415)
[1] -1
>exp(0)
[1] 1
>log(1)
[1] 0

The results above are displayed but not stored. To store a result, we give it a name. In other words, we assign the result to a variable. Once we have done that, we can refer to it later by using the variable name.

Assigning a result in a variable is done using <-. For example:

>five <- 3+2
> five
[1] 5
> five + 1
[1] 6
Note

Note that when we write five at the R command prompt, R prints the value stored in that variable. If we want the exact text five, we have to put it in quotes:

> "five" 
[1] "five"

In R it does not matter whether you use single or double quotes, as long as they match. Working with text values will be discussed further in the next section.

One advantage of using R is that we can do calculations on multiple values, or vectors, at once. For example:

> c(1, 2)  + c(1, 3) 
[1] 2 5

This is very convenient when we work with a data set containing many observations and want to do the same calculation for each of them. We will return to this idea later.

How we interact with R

Working in R is conceptually simple: we type commands and ask R to execute them with the ENTERENTER key.

You can type commands directly into the console, but once your work becomes longer than a few lines this quickly becomes inconvenient. For that reason, most serious work in R is written in a script file. A script gives you a record of what you did, allows you to rerun your work, and makes it much easier to share your analysis with someone else.

Since R code is just text, you can write it in any text editor. In practice, the easiest and most efficient way to work is to use an integrated development environment (IDE), such as RStudio, where the editor, console, files and plots are all visible together.

RStudio: the easiest way to work with R

For most users, especially those new to R, RStudio is probably the easiest IDE to start with. Like everything about R, you are free to choose whichever editor you like. Let’s have a look at an RStudio session.

There are four panes:

  • The top left is the source pane. Do most of your typing in this pane.
  • The bottom left is the console. This is where you can see what R is doing and where output appears.
  • The top right is the environment pane, which shows the objects currently stored in memory.
  • The bottom right usually contains files, plots, packages and help.

Type your commands in the source pane and send them to R using the keyboard shortcut . This sends the current line, or the selected code, to the console. Open RStudio on your computer and try it now. Alternatively, you can press the Run button below the file tabs.

Because programming involves a lot of typing, keyboard shortcuts save a lot of time. A useful reference is the collection of RStudio cheatsheets.

A few other things about RStudio:

  • move the focus between panes using the keyboard shortcuts Ctrl+1Ctrl+1 and Ctrl+2Ctrl+2 for the source and console panes respectively
  • the little asterisk next to your source code file name indicates unsaved code
  • save your source code!

R Packages

Base R has many functions, but many more come from extension packages. These can be installed using install.packages():

install.packages("packageName",
                 lib = "/directory/to/my custom R library",
                 repos = "http://cran.xl-mirror.nl")

# usually lib and repos can be omitted (left at the default)

The package name must be quoted when installing.

Later, when you want to use the package, you have to load it first using library():

library("packageName")      ## quotes are optional when loading a package
Note

You have to install a package only once, but you have to load it every time you start a new R session and want to use it. Usually somewhere at the top of a script, you will find a section with these library commands so it is clear which packages are needed to run in. It is considered bad practice to use install.packages() inside a script because it may install packages without the user’s consent.

Comments

To make your scripts clearer, you can add comments to explain what the code is doing and why it is doing this. Text that represents a comment is indicated by the hash (#) symbol. For example:

# load the survival package because we will need the data sets
library(survival)

The first line does not actually do anything because it is skipped by the interpreter. However, it explains why the library() call on the next line is included. It is also possible to add comments at the end of a line of code:

library(survival) # for data sets

Backslashes

One more thing worth mentioning is how R interprets backslashes and forward slashes, that is \\ and /, when you specify the location of files on your computer.

On Unix-like systems such as Linux and macOS, the normal separator in file paths is the forward slash (/). Windows often displays file paths using backslashes (\), but in R you can usually still use forward slashes on Windows as well. That is often the simplest option:

myfile <- "C:/Users/name/Documents/data.csv"

If you do use backslashes in a character string, you usually need to double them:

myfile <- "C:\\Users\\name\\Documents\\data.csv"

The reason is that in R strings, the backslash is also used as an escape character. For example, \n means a new line and \t means a tab. If you want an actual backslash character inside a string, you normally have to write it as \\.

For beginners, the easiest rule is: when typing file paths in R, prefer forward slashes whenever possible.1

Getting Help

  • Online help. This is usually the first place to look. It also works when you are actually offline:
?help
help(help)
??help
apropos("help")              # quotes are needed

Try them out and get used to reading help pages early.

  • Google. When you are actually online, this is often where you go when you have no idea where to start: Google.

  • Package vignettes. When the regular help page does not provide enough context, a vignette often gives a longer explanation with examples. For example, see the limma vignette: http://www.bioconductor.org/packages/release/bioc/vignettes/limma/inst/doc/usersguide.pdf

  • Mailing lists. R-help: http://www.r-project.org/mail.html. Before posting, check with Google or searchable archives whether your question has already been asked.

  • Other lists: https://stat.ethz.ch/mailman/listinfo

  • Stack Overflow: https://stackoverflow.com. There are many R questions and answers there, often with practical examples.

Trouble shooting

  • Clean your environment and start again, so that you can be sure a messy workspace is not the cause of your issue. In RStudio, click the broom in the Environment tab; in any R session you can also run:
rm(list=ls())                 
ls()           # quotes are needed

If the above does not work, try these:

  • Start R without any customisations, i.e., omit loading the .Rprofile and .Renviron customisation files. From the command line:
R --vanilla
  • trace.back() function: very helpful pinpointing the source of an error, and thus its cause.

Check yourself

  1. What is the difference between typing code in the source pane and typing it in the console?
  2. What is the difference between installing a package and loading a package?
  3. How would you ask R for help on a function you do not understand?
